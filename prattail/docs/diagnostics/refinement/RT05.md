# RT05: refinement-decidability-tier

**Severity:** Note
**Module:** Refinement Type System
**Feature gate:** `type-system`

## What It Detects

Reports the decidability tier (T1–T4) of a refinement type's predicate, which
determines how the predicate is evaluated at runtime:

| Tier | Description | Runtime Cost | Example |
|------|-------------|--------------|---------|
| T1 | Statically decidable | O(0) — eliminated | `true`, `false` |
| T2 | Runtime decidable | O(1) to O(\|value\|) | `x > 0`, `safe(x)` |
| T3 | Bounded runtime | O(k · \|value\|) | `exists_{k=100} y. P(x, y)` |
| T4 | User-asserted | O(1) trust | Undecidable predicates with proof |

Classification follows the same `classify_decidability()` pipeline used for
behavioral guard predicates (§11 of predicated-types.md).

## Example

### Triggering Code

```
types {
    Int:i64 { ... }
    PosInt = { x: Int | x > 0 };
}
```

The predicate `x > 0` is a single-variable linear inequality — classified as
T2 (runtime decidable via inline arithmetic).

### Diagnostic Output

```
note[RT05] (MyLang): refinement type 'PosInt' predicate decidability: T2 (runtime decidable)
  = hint: this predicate will be evaluated inline at O(1) per tuple
```

### What Each Tier Means for Codegen

- **T1**: Predicate is evaluated at compile time. If true, the refinement is
  tautological (see RT02). If false, the refinement is unsatisfiable (see RT01).
  No runtime code generated.
- **T2**: Predicate compiled to inline check (arithmetic comparison, relation
  lookup, register automaton, or SFA transition table). Generated as `if { ... }`
  guard in the Ascent population rule.
- **T3**: Predicate compiled to bounded iteration with depth counter. May return
  Unknown, in which case the value is conservatively excluded.
- **T4**: User provides an `assert_pred()` annotation or Rocq proof. Generated
  code trusts the assertion.

## Rationale

- Decidability tier affects both compile-time analysis capability and runtime
  performance. T1 predicates are free; T2 are fast; T3 have bounded cost; T4
  require external proof. Knowing the tier helps the grammar author optimize
  predicate complexity.
- This lint is informational — it does not indicate a problem, just a
  classification result.
