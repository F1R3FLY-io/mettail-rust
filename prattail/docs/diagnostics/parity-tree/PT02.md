# PT02: pata-subsumption

**Severity:** Note
**Module:** M5 Parity Alternating Tree Automata
**Feature gate:** `parity-tree-automata`

## What It Detects

- A PATA has states but all states have trivial parity (max_priority = 0). This means the parity condition degenerates to a simple reachability or safety check -- the full power of parity acceptance (alternating least/greatest fixpoints) is unused.
- The automaton could be simplified to a non-parity tree automaton, reducing analysis complexity.

## Example

### Triggering Code

```
language! {
    name: SimpleTree;
    primary: Expr;

    category Expr {
        Add = Expr "+" Expr;
        Lit = <int>;
    }

    // Predicate that only uses greatest fixpoint (nu), no alternation:
    // "every leaf is a literal"
    predicate all_leaves_lit: nu X. (label = Lit) \/ ([] X);
}
```

The compiled PATA assigns priority 0 to all states because there is only a single `nu` fixpoint (even priority) and no `mu` (odd priority).

### Diagnostic Output

```
note[PT02] (SimpleTree): PATA has 3 states but max_priority=0 -- all states have trivial parity
  = hint: consider simplifying to a non-parity tree automaton
```

### Fix

```
// The predicate is correct but uses more machinery than needed.
// A standard tree automaton (WTA) suffices for non-alternating properties:
// Use tree_automaton analysis directly instead of parity-tree-automata.

// If alternating fixpoints are genuinely needed, the formula should
// contain both mu and nu operators:
predicate complex: nu X. mu Y. (label = Add /\ <1> X /\ <2> Y) \/ (label = Lit);
```

## Rationale

- The lint fires when `result.num_states > 0 && result.max_priority == 0`. A max priority of 0 means only even priorities are used (all greatest fixpoints), which corresponds to a Buchi/safety condition rather than a full parity condition.
- Zielonka's algorithm complexity is O(n^d x m) where `d` is the priority depth. With `d = 0`, the algorithm degenerates to BFS reachability, but the PATA framework still carries overhead for priority tracking.
- The existing `tree_automaton.rs` module provides simpler WTA analysis for non-parity properties. Using it directly avoids the PATA compilation overhead.
- Subsumption of the parity condition suggests the mu-calculus formula does not fully utilize fixpoint alternation. This is informational -- the analysis is still correct, just potentially suboptimal.
