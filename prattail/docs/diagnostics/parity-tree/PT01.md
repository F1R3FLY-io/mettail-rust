# PT01: pata-emptiness-violation

**Severity:** Warning
**Module:** M5 Parity Alternating Tree Automata
**Feature gate:** `parity-tree-automata`

## What It Detects

- A parity alternating tree automaton (PATA) constructed from a mu-calculus predicate formula is empty -- no AST (abstract syntax tree) can satisfy the predicate.
- This means the predicate is vacuously false: no parse tree in the grammar's language matches the specified structural property. The predicated receive or verification check will never succeed.

## Example

### Triggering Code

```
language! {
    name: TreeCheck;
    primary: Expr;

    category Expr {
        Add = Expr "+" Expr;
        Mul = Expr "*" Expr;
        Neg = "-" Expr;
        Lit = <int>;
    }

    // Mu-calculus predicate: "every node is both Add and Mul simultaneously"
    // This is unsatisfiable because a node cannot have two different labels.
    predicate all_add_mul: nu X. (label = Add) /\ (label = Mul) /\ [] X;
}
```

### Diagnostic Output

```
warning[PT01] (TreeCheck): parity tree automaton is empty -- no AST can match this predicate
  = hint: check that the mu-calculus formula is not vacuously false
```

### Fix

```
// Fix the predicate to express the actual intent:
// "every node is either Add or Mul"
predicate all_add_or_mul: nu X. ((label = Add) \/ (label = Mul)) /\ [] X;

// Or: "every branch has at least one Add"
predicate has_add: mu X. (label = Add) \/ <> X;
```

## Rationale

- PATA emptiness is checked via Zielonka's recursive parity game algorithm in O(n^d x m) where `n` is the state count, `d` is the priority depth, and `m` is the transition count. An empty PATA means the Attacker wins from every initial position.
- The mu-calculus to PATA compilation maps: mu X to odd priority (least fixpoint), nu X to even priority (greatest fixpoint), diamond <> to existential branching, box [] to universal branching. Vacuously false formulas produce automata where no tree can satisfy all universal constraints simultaneously.
- In predicated types, an empty PATA on a receive guard means no incoming message can ever pass the guard. This is the tree-level analogue of SYM01 (unsatisfiable guard on flat words).
- Empty PATAs waste compile-time resources (the emptiness check itself is expensive) and indicate a logic error in the predicate specification.
