# PT03: pata-high-priority

**Severity:** Note
**Module:** M5 Parity Alternating Tree Automata
**Feature gate:** `parity-tree-automata`

## What It Detects

- A PATA has priority depth exceeding 4, meaning the mu-calculus formula uses deeply nested alternating fixpoints (mu inside nu inside mu, etc.).
- Emptiness checking for parity tree automata has complexity O(n^d x m) where `d` is the priority depth. With `d > 4`, the emptiness check becomes exponentially expensive in practice.

## Example

### Triggering Code

```
language! {
    name: DeepFixpoint;
    primary: Proc;

    category Proc {
        Send = "!" Chan Proc;
        Recv = "?" Chan Proc;
        Par  = Proc "|" Proc;
        Zero = "0";
    }

    // Deeply nested fixpoints (priority depth 5):
    predicate deep:
        nu X1. mu X2. nu X3. mu X4. nu X5.
            (label = Send /\ <1> X2)
            \/ (label = Recv /\ <1> X3)
            \/ (label = Par /\ <1> X4 /\ <2> X5)
            \/ (label = Zero /\ X1);
}
```

### Diagnostic Output

```
note[PT03] (DeepFixpoint): PATA priority depth 5 > 4 -- exponential blowup in emptiness checking
  = hint: reduce fixpoint nesting to improve analysis performance
```

### Fix

```
// Flatten the fixpoint structure by decomposing into simpler properties:
predicate reachability: mu X. (label = Zero) \/ <> X;
predicate liveness: nu Y. mu Z. (label = Send) \/ <> Z;

// Or reduce alternation depth by combining fixpoints:
predicate simplified:
    nu X. mu Y.
        (label = Send /\ <1> Y)
        \/ (label = Zero /\ X);
```

## Rationale

- The lint fires when `result.priority_depth > 4`. Each level of fixpoint alternation increases the priority index by 1. Zielonka's algorithm solves the emptiness game by recursive decomposition over priority levels; each level multiplies the state space by O(n).
- With priority depth `d`:
  - `d = 1`: Safety/reachability (linear time)
  - `d = 2`: Buchi/co-Buchi (quadratic)
  - `d = 3`: Rabin/Streett (cubic)
  - `d = 4`: Practical limit for most grammars (quartic)
  - `d > 4`: Exponential blowup risk (O(n^d) with d > 4)
- In the mu-calculus, alternation depth corresponds to the complexity of the temporal property being verified. Most grammar invariants can be expressed with depth 2-3 (safety + liveness).
- The Emerson-Jutla construction (mu-calculus to PATA) preserves alternation depth, so reducing the formula's nesting directly reduces the PATA's priority depth.
