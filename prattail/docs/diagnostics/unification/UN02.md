# UN02: tautological-unification-guard

**Severity:** Warning
**Module:** Unification
**Feature gate:** `unification`

## What It Detects

A unification guard is trivially satisfiable -- the pattern matches any input term.
This occurs when both sides of the unification equation are bare variables (or
structurally equivalent variable patterns), so any substitution satisfies the
constraint. The guard is redundant: removing it does not change runtime behavior.

Detection runs the standard Robinson unification algorithm. If the most general
unifier (MGU) maps every variable to a fresh variable (or to itself), the guard
imposes no structural constraint on the input.

## Example

### Triggering Code

```
language! {
    name: Router;
    primary: Proc;

    category Proc {
        // x ≡ y is satisfied by any substitution {x ↦ t, y ↦ t}
        CatchAll = "for" "(" "@t" ":" "x = y" "<-" ch ")" "{" Proc "}";
        Zero = "0";
    }
}
```

The guard `x = y` is satisfied by setting `x` and `y` to the same term -- any
input satisfies this.

### Diagnostic Output

```
warning[UN02] (Router): unification guard 'x = y' is tautological (redundant)
  = note: guard matches any input and can be removed
```

### Fix

```
// Remove the redundant guard -- the receive already accepts all terms.
CatchAll = "for" "(" "@t" "<-" ch ")" "{" Proc "}";

// Or add meaningful structural constraints:
Structured = "for" "(" "@t" ":" "f(x, a) = f(y, a)" "<-" ch ")" "{" Proc "}";
```

## Rationale

- A tautological unification guard consumes runtime resources (unification
  algorithm execution) without filtering any inputs. While the cost is small
  per-invocation, it accumulates in high-throughput message dispatch scenarios.
- The tautology check is a byproduct of computing the MGU: if the resulting
  substitution is a variable renaming (a permutation on variable names), the
  guard is tautological. This check runs in O(1) additional time after
  unification completes.
- Tautological guards often indicate that the grammar author intended to add
  structural constraints but left a placeholder pattern in place.
