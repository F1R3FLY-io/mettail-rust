# UN01: unsatisfiable-unification-guard

**Severity:** Warning
**Module:** Unification
**Feature gate:** `unification`

## What It Detects

A structural pattern guard fails unification -- the guard specifies a term equation
that has no solution. This occurs due to:

1. **Constructor clash:** The left-hand side and right-hand side have different
   head constructors (e.g., `f(...)` vs `g(...)`), and no substitution can make
   them equal.
2. **Occurs check failure:** A variable `x` appears on one side and the other side
   contains a term with `x` as a proper sub-term (e.g., `x ≡ f(x)`), which would
   require an infinite term.

In either case, the guarded rule can never match -- it is dead code.

## Example

### Triggering Code

```
language! {
    name: TypeCheck;
    primary: Proc;

    category Proc {
        // Constructor clash: f ≠ g, so f(a, x) can never equal g(b, y)
        BadMatch = "for" "(" "@t" ":" "f(a, x) = g(b, y)" "<-" ch ")" "{" Proc "}";

        // Occurs check: x cannot equal f(x) in finite terms
        Infinite = "for" "(" "@t" ":" "x = f(x)" "<-" ch ")" "{" Proc "}";

        Zero = "0";
    }
}
```

### Diagnostic Output

```
warning[UN01] (TypeCheck): unification guard 'f(a, x) = g(b, y)' is unsatisfiable (constructor clash: f ≠ g)
  = hint: remove the unreachable guard or fix the structural pattern

warning[UN01] (TypeCheck): unification guard 'x = f(x)' is unsatisfiable (occurs check failure)
  = hint: remove the unreachable guard or fix the structural pattern
```

### Fix

```
// Fix the constructor clash by using matching constructors:
Match = "for" "(" "@t" ":" "f(a, x) = f(b, y)" "<-" ch ")" "{" Proc "}";

// Fix the occurs check by avoiding self-reference:
Finite = "for" "(" "@t" ":" "x = f(y)" "<-" ch ")" "{" Proc "}";
```

## Rationale

- Unification failure is decidable in near-linear time (using union-find with path
  compression and union by rank). The occurs check adds at most a linear scan of
  the right-hand term.
- Constructor clash is the most common cause of UN01: it typically indicates a
  copy-paste error where the grammar author intended matching constructors.
- In typed process calculi, an unsatisfiable unification guard on a channel means
  all messages on that channel are silently dropped by the guarded receive,
  potentially causing deadlock.
- This is the structural analogue of SYM01 (unsatisfiable symbolic guard) and PB01
  (unsatisfiable arithmetic guard). All three detect dead receives through different
  decision procedures.
