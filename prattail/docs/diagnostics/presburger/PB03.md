# PB03: subsumed-arithmetic-guard

**Severity:** Note
**Module:** Presburger Arithmetic
**Feature gate:** `presburger`

## What It Detects

One arithmetic guard's satisfying set is a subset of another's: for guards A and B
on the same channel, every integer assignment satisfying A also satisfies B, i.e.,
`Sol(A) ⊆ Sol(B)`. The subsumed guard (A) is redundant when both guards appear on
the same channel -- any input matching A also matches B, so A adds no
distinguishing power.

Detection constructs `NFA(A) ∩ NFA(complement(B))` and checks for emptiness. If
this intersection is empty, every accepting run of A is also an accepting run of B,
confirming subsumption.

## Example

### Triggering Code

```
language! {
    name: Filter;
    primary: Proc;

    category Proc {
        Tight = "for" "(" "@x" ":" "x <= 3" "<-" ch ")" "{" TightProc "}";
        Loose = "for" "(" "@x" ":" "x <= 5" "<-" ch ")" "{" LooseProc "}";
        Zero  = "0";
    }
}
```

The guard `x <= 3` is subsumed by `x <= 5`: every integer satisfying `x <= 3` also
satisfies `x <= 5`.

### Diagnostic Output

```
note[PB03] (Filter): arithmetic guard 'x <= 3' is subsumed by 'x <= 5' (redundant)
  = hint: the subsuming guard already covers this constraint
```

### Fix

```
// Option 1: Remove the subsumed guard; handle all values with the broader one.
Loose = "for" "(" "@x" ":" "x <= 5" "<-" ch ")" "{" LooseProc "}";

// Option 2: Make guards disjoint to retain both branches.
Tight = "for" "(" "@x" ":" "x <= 3"           "<-" ch ")" "{" TightProc "}";
Wide  = "for" "(" "@x" ":" "x > 3 AND x <= 5" "<-" ch ")" "{" WideProc  "}";
```

## Rationale

- Subsumption implies overlap (Sol(A) ⊆ Sol(B) means Sol(A) ∩ Sol(B) = Sol(A) ≠ ∅
  unless A is unsatisfiable). When the runtime evaluates both guards, the subsumed
  guard's branch is always shadowed by the subsuming guard's branch under
  priority-ordered dispatch.
- The subsumption check is a refinement of PB01/PB02: if Sol(A) ⊆ Sol(B) and
  Sol(A) ≠ ∅, neither guard is individually unsatisfiable or tautological, but
  their combination has redundancy.
- The NFA intersection + complement + emptiness pipeline runs in O(|Q_A| * |Q_B|)
  states with a constant factor from the binary encoding of variable domains.
