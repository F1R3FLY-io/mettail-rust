# PB01: unsatisfiable-arithmetic-guard

**Severity:** Warning
**Module:** Presburger Arithmetic
**Feature gate:** `presburger`

## What It Detects

A linear arithmetic guard (conjunction/disjunction of integer linear inequalities)
is provably unsatisfiable via Presburger automata NFA emptiness. The guard's NFA
accepts no input -- there exists no integer assignment to the guard's free variables
that satisfies all constraints simultaneously. The guarded rule is dead code: it
can never fire at runtime.

Detection uses the automata-based Presburger decision procedure: each atomic
inequality is compiled to an NFA over the binary encodings of variable values,
Boolean connectives are realized by NFA product/union/complement, and the final
automaton is checked for emptiness via BFS reachability from the initial state to
any accepting state.

## Example

### Triggering Code

```
language! {
    name: Scheduler;
    primary: Proc;

    category Proc {
        HighPriority = "for" "(" "@x" ":" "x + y <= 10 AND x >= 15" "<-" ch ")" "{" Proc "}";
        LowPriority  = "for" "(" "@x" "<-" ch ")" "{" Proc "}";
        Zero = "0";
    }
}
```

The guard `x + y <= 10 AND x >= 15` requires `x >= 15`, which forces
`y <= 10 - x <= 10 - 15 = -5`. For non-negative `y` this is impossible; more
generally, the Presburger NFA for the conjunction is empty regardless of the
domain.

### Diagnostic Output

```
warning[PB01] (Scheduler): arithmetic guard 'x + y <= 10 AND x >= 15' is unsatisfiable (dead rule)
  = hint: remove the unreachable rule or relax the numeric constraint
```

### Fix

```
// Option 1: Relax the constraint so it has a satisfying region.
HighPriority = "for" "(" "@x" ":" "x + y <= 20 AND x >= 15" "<-" ch ")" "{" Proc "}";

// Option 2: Remove the dead rule entirely.
```

## Rationale

- An unsatisfiable arithmetic guard wastes compile-time resources constructing
  automaton states that can never contribute to runtime dispatch. It also hides
  potential logic errors: the grammar author likely intended a satisfiable
  constraint region.
- The Presburger automaton construction runs in time exponential in the number of
  quantifier alternations but polynomial for quantifier-free formulas (the common
  case for guards). Emptiness checking is linear in the NFA state count.
- In process calculi with guarded receives, an unsatisfiable guard silently drops
  all messages on the guarded channel, potentially causing deadlock or starvation.
