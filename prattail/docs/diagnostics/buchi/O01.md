# O01: weighted-buchi-non-convergent

**Severity:** Warning
**Module:** M2 Weighted Buchi Automata
**Feature gate:** `omega`

## What It Detects

- A weighted Buchi automaton with more than one state has no accepting cycle, meaning the total accepting weight computation trivially converges to zero.
- This indicates that the liveness property encoded by the automaton can never be satisfied on infinite input -- no infinite run visits an accepting state infinitely often.

## Example

### Triggering Code

```
language! {
    name: StreamParser;
    primary: Token;

    category Token {
        Data    = <ident>;
        Delim   = ",";
        Term    = ";";
    }

    // Liveness property: "data tokens appear infinitely often"
    // But the Buchi automaton constructed has no accepting cycle
    // because all paths from accepting states lead to dead ends.
    liveness progress: eventually_always(Data);
}
```

### Diagnostic Output

```
warning[O01] (StreamParser): weighted Buchi automaton (5 states) has no accepting cycle -- weight computation trivially converges to zero
  = hint: check that liveness properties are correctly specified
```

### Fix

```
// Ensure the Buchi automaton has a reachable accepting cycle.
// Typically this means the liveness formula should reference
// states that can loop:
liveness progress: always_eventually(Data);

// Or restructure the grammar so accepting states are reachable
// from themselves via at least one cycle.
```

## Rationale

- A Buchi automaton accepts an infinite word if and only if some accepting state is visited infinitely often. If no accepting cycle exists (`!result.has_accepting_cycle`), the language is empty -- no infinite word is accepted.
- The weighted variant computes `total_accepting_weight()` via Tarjan SCC decomposition followed by `matrix_star()` for cycle weights. Without an accepting SCC, this computation yields the zero element of the semiring.
- This lint fires when `!result.has_accepting_cycle && result.num_states > 1`. The `num_states > 1` guard avoids false positives on trivial single-state automata (which may intentionally accept nothing).
- In stream/REPL parsing, a non-convergent Buchi automaton means the progress guarantee cannot be verified -- the parser may stall without detection.
