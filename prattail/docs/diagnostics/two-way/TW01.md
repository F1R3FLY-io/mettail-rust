# TW01: circular-channel-dependency

**Severity:** Warning
**Module:** M11 Weighted Two-Way Transducers
**Feature gate:** `two-way-transducer`

## What It Detects

- Bidirectional reachability analysis via the weighted two-way transducer detects a circular dependency cycle among channels: channel A waits for input from channel B, while channel B waits for input from channel A (directly or transitively).
- Circular channel dependencies cause deadlock at runtime: no channel can proceed because each is blocked waiting for another channel in the cycle.

## Example

### Triggering Code

```
language! {
    name: Deadlock;
    primary: Proc;

    category Proc {
        Par  = Proc "|" Proc;
        Send = Chan "!" "(" Expr ")";
        Recv = "for" "(" "@x" "<-" Chan ")" "{" Proc "}";
        Zero = "0";
    }

    // Process 1: receives on ch_a, then sends on ch_b
    // Process 2: receives on ch_b, then sends on ch_a
    // Circular dependency: ch_a -> ch_b -> ch_a
    example deadlock_proc:
        (for (@x <- ch_a) { ch_b!("hello") })
        | (for (@y <- ch_b) { ch_a!("world") });
}
```

### Diagnostic Output

```
warning[TW01] (Deadlock): circular channel dependency detected: ch_a -> ch_b -> ch_a
  = hint: break the circular dependency to prevent deadlock
```

### Fix

```
// Option 1: Break the cycle by providing an initial value:
example fixed_proc:
    ch_a!("init")
    | (for (@x <- ch_a) { ch_b!(x) })
    | (for (@y <- ch_b) { ch_a!(y) });

// Option 2: Use a mediator channel:
example mediated_proc:
    (for (@x <- ch_a) { ch_med!(x) })
    | (for (@m <- ch_med) { ch_b!(m) })
    | (for (@y <- ch_b) { done!("ok") });
```

## Rationale

- The W2T's `detect_deadlock(processes)` operation performs conjunctive bidirectional reachability in O(|P|^2 x |channels|). The forward pass builds a channel dependency graph; the backward pass from accepting states identifies which initial states must participate. Cycles in this graph represent deadlocks.
- Unlike WPDS analysis (forward-only) or forward-backward analysis (DAG-only, no cycle support), the W2T can detect circular dependencies because its bidirectional head movement naturally models the "wait-for" relationship between channels.
- Each cycle in `result.deadlock_cycles` is reported as a chain of channel names (e.g., `["ch_a", "ch_b", "ch_a"]`). The cycle is guaranteed to cause deadlock under synchronous communication semantics.
- In the Feng & Maletti (2022) formulation, deadlock cycles correspond to crossing sequences that never reach the right endmarker -- the transducer loops indefinitely between forward and backward states.
