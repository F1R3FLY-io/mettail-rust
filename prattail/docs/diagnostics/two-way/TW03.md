# TW03: constraint-propagation-divergent

**Severity:** Warning
**Module:** M11 Weighted Two-Way Transducers
**Feature gate:** `two-way-transducer`

## What It Detects

- The backward constraint propagation phase of a weighted two-way transducer may diverge: the transducer has backward states (`num_backward > 0`) and the dependency analysis detected deadlock cycles among channels.
- When backward constraint propagation encounters a cycle, it can refine constraints indefinitely without reaching a fixpoint, consuming unbounded time and memory.

## Example

### Triggering Code

```
language! {
    name: CrossChannel;
    primary: Proc;

    category Proc {
        Par  = Proc "|" Proc;
        Recv = "for" "(" "@x" "<-" Chan ")" "{" Proc "}";
        Send = Chan "!" "(" Expr ")";
    }

    // Two-way transducer with backward constraint propagation
    // that encounters a circular dependency:
    two_way_constraint cross_check {
        // Forward pass: decompose pattern on ch_a
        forward(ch_a) -> extract(x, y);
        // Backward pass: propagate constraint to ch_b
        backward(ch_b) -> constrain(x > 0);
        // But ch_b's output feeds back into ch_a via another process,
        // creating a propagation cycle: ch_a -> ch_b -> ch_a
    }
}
```

### Diagnostic Output

```
warning[TW03] (CrossChannel): backward constraint propagation may diverge: 2 backward states with 1 deadlock cycles
  = hint: add a propagation depth limit or break the deadlock cycles
```

### Fix

```
// Option 1: Add a propagation depth limit:
two_way_constraint cross_check {
    max_propagation_depth: 10;
    forward(ch_a) -> extract(x, y);
    backward(ch_b) -> constrain(x > 0);
}

// Option 2: Break the circular dependency (see TW01):
// Ensure backward propagation does not feed back into itself.

// Option 3: Use one-way constraint checking instead:
// If backward propagation is not essential, switch to forward-only.
constraint cross_check {
    forward(ch_a) -> extract(x, y);
    forward(ch_b) -> check(x > 0);
}
```

## Rationale

- The lint fires when `result.num_backward > 0 && !result.deadlock_cycles.is_empty()`. Backward states alone are safe; deadlock cycles alone may cause runtime deadlock but not analysis divergence. The combination -- backward propagation through a cycle -- is what causes analysis divergence.
- The `backward_constraint(output, channel)` operation inverts the transducer: given a desired output, it computes input constraints in O(|output| x |Q|^2). When the output feeds back into the input via a cycle, this inversion is applied repeatedly, with each iteration potentially refining the constraint further.
- Without a propagation depth limit, the refinement loop can produce an infinite sequence of increasingly precise constraints, none of which is a fixpoint. This is analogous to infinite regression in fixpoint computation.
- In the Feng & Maletti (2022) theory, convergence of crossing sequences is guaranteed for finite input. However, when the input itself is computed from the output (circular dependency), the effective input is unbounded, breaking the convergence guarantee.
- A propagation depth limit truncates the refinement after k iterations, yielding an overapproximation of the true constraint. This trades precision for termination.
