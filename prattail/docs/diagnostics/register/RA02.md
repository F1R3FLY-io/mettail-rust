# RA02: redundant-register

**Severity:** Note
**Module:** M6 Register Automata
**Feature gate:** `register-automata`

## What It Detects

- A register in the register automaton is written (via a `Store` operation) but never tested (no `TestEq`, `TestNeq`, or `TestFresh` references it on any reachable transition).
- The register consumes storage and adds complexity to the automaton without influencing any dispatch decision. It can be safely removed to simplify the automaton.

## Example

### Triggering Code

```
language! {
    name: Binding;
    primary: Stmt;

    category Stmt {
        Let = "let" <ident> "=" Expr;
        Use = <ident>;
    }

    context_rule track_bindings {
        // r0: stores the bound identifier (used for later matching)
        // r1: stores the expression type (NEVER tested anywhere)
        transition(Let -> Body, store: r0 = ident, store: r1 = Expr);
        transition(Body -> Use, test: r0 == ident);
        // r1 is stored but never referenced in any test
    }
}
```

### Diagnostic Output

```
note[RA02] (Binding): register 1 is written but never tested
  = hint: remove the unused register to simplify the automaton
```

### Fix

```
// Remove the unused register:
context_rule track_bindings {
    transition(Let -> Body, store: r0 = ident);
    transition(Body -> Use, test: r0 == ident);
}
```

## Rationale

- The normalization algorithm in register automata identifies dead registers by scanning all transitions for `TestEq`, `TestNeq`, and `TestFresh` operations. A register index that appears only in `Store` operations and never in any test operation is classified as dead.
- Dead registers increase the automaton's state space during simulation: each register adds a dimension to the register valuation, multiplying the effective state count by the domain size.
- Eliminating dead registers is a safe optimization: the automaton's accepted language is unchanged because the register's value never influences any transition guard. The normalization complexity is O(V x k x E) where V is the state count, k is the register count, and E is the transition count.
- In data-dependent dispatch (extending `decision_tree.rs`), unused registers create unnecessary dispatch branches that can never be distinguished from their alternatives.
