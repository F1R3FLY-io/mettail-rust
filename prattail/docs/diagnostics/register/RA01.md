# RA01: unbound-data-reference

**Severity:** Warning
**Module:** M6 Register Automata
**Feature gate:** `register-automata`

## What It Detects

- A transition in the register automaton tests a register (via `TestEq`, `TestNeq`, or `TestFresh`) that has no reachable `Store` operation on any path from an initial state to that transition.
- Reading an unbound register produces undefined behavior in the data-aware dispatch: the comparison has no meaningful value to compare against, making the transition's guard vacuously true or false depending on the initial register state.

## Example

### Triggering Code

```
language! {
    name: TagMatch;
    primary: Elem;

    category Elem {
        Open  = "<" Tag ">";
        Close = "</" Tag ">";
        Text  = <string>;
    }

    category Tag {
        Name = <ident>;
    }

    // Context-sensitive rule: closing tag must match opening tag.
    // Register r0 should store the opening tag, but the Store
    // is missing on some paths.
    context_rule match_tags {
        transition(Open -> Content, store: r0 = Tag);
        transition(Content -> Close, test: r0 == Tag);   // OK path
        transition(Open -> Close, test: r0 == Tag);       // BUG: no Store on this path
    }
}
```

### Diagnostic Output

```
warning[RA01] (TagMatch): transition 2 tests register 0 which has no reachable Store operation
  = hint: ensure the register is stored before being tested
```

### Fix

```
// Ensure every path to a TestEq/TestNeq transition passes through
// a Store for the tested register:
context_rule match_tags {
    transition(Open -> Content, store: r0 = Tag);
    transition(Content -> Close, test: r0 == Tag);
    // Remove the direct Open -> Close transition, or add a Store:
    transition(Open -> Close, store: r0 = Tag, test: r0 == Tag);
}
```

## Rationale

- The analysis performs reachability from initial states to each transition that contains a `TestEq`, `TestNeq`, or `TestFresh` operation. For each such transition, it checks whether every path to it includes a `Store` for the tested register index.
- Unbound register references are analogous to use-before-initialization in imperative languages. The register's initial value is `None` (no stored data), so a `TestEq` against `None` will always fail and a `TestFresh` will always succeed -- neither outcome reflects the programmer's intent.
- In context-sensitive parsing, unbound registers can cause the parser to accept or reject inputs incorrectly. For example, an unbound register in a tag-matching rule would accept any closing tag regardless of the opening tag.
- The complexity of unbound reference detection is O(|word| x |states| x k!) where k is the number of registers, dominated by the nondeterministic simulation.
