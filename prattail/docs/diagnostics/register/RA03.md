# RA03: register-equivalence

**Severity:** Note
**Module:** M6 Register Automata
**Feature gate:** `register-automata`

## What It Detects

- All registers in the register automaton are dead (written but never tested), meaning the entire automaton is equivalent to a plain finite automaton (FA).
- The register machinery adds no value: no data-dependent decisions are being made. The automaton can be replaced by a standard FA with lower complexity.

## Example

### Triggering Code

```
language! {
    name: Tracker;
    primary: Token;

    category Token {
        Kw   = "let" | "in" | "fn";
        Ident = <ident>;
        Op   = "+" | "-" | "*";
    }

    context_rule stateful_lex {
        // Stores values but never tests them:
        transition(Start -> KwSeen, store: r0 = Kw);
        transition(KwSeen -> IdSeen, store: r1 = Ident);
        transition(IdSeen -> OpSeen, store: r2 = Op);
        // All transitions use Nop for register tests -- no TestEq/TestNeq
    }
}
```

### Diagnostic Output

```
note[RA03] (Tracker): all 3 registers are dead -- automaton is equivalent to a plain FA
  = hint: consider using a standard FA instead of a register automaton
```

### Fix

```
// If data-dependent dispatch is not needed, use a plain FA:
context_rule stateful_lex {
    transition(Start -> KwSeen);
    transition(KwSeen -> IdSeen);
    transition(IdSeen -> OpSeen);
}

// Or, if data tracking was intended, add tests:
context_rule stateful_lex {
    transition(Start -> KwSeen, store: r0 = Kw);
    transition(KwSeen -> IdSeen, store: r1 = Ident);
    transition(IdSeen -> End, test: r0 == "let", test: r1 == Ident);
}
```

## Rationale

- The lint fires when `result.num_registers > 0 && result.dead_registers.len() == result.num_registers`. This is the extreme case of RA02 where every single register is dead.
- A register automaton with all-dead registers has the same expressiveness as an NFA: it can be reduced to an NFA via the `normalize()` operation (eliminate dead registers), yielding an automaton with 0 registers.
- The NFA equivalence matters for complexity: register automaton simulation is O(|word| x |states| x k!) due to nondeterministic register valuations. With k = 0, this reduces to O(|word| x |states|) -- standard NFA simulation.
- This lint suggests either (a) the register automaton was created by a template/macro that always allocates registers, even when unused, or (b) the data-dependent logic was removed during refactoring but the register infrastructure was not cleaned up.
- The reduction to `nominal.rs` (orbit-finite bisimulation) is trivially sound when all registers are dead: the orbit structure collapses to the state structure.
