# SL01: unsatisfiable-subtype-constraint

**Severity:** Warning
**Module:** Subtype Lattice
**Feature gate:** `lattice-theory`

## What It Detects

A subtype constraint set contradicts the declared type hierarchy. The constraint
solver determines that no valid type assignment exists -- the conjunction of
subtyping constraints is unsatisfiable given the lattice structure.

This typically occurs when:

1. **Cyclic incompatibility:** Two types are constrained to be mutual subtypes
   (`A ≤ B` and `B ≤ A`) but are declared as distinct, incompatible types in the
   lattice (no antisymmetric collapse).
2. **Glb/Lub failure:** A type variable is constrained to be a subtype of two
   types whose greatest lower bound (glb) is bottom (⊥), making the constraint
   unsatisfiable.
3. **Lattice gap:** A required subtype relationship traverses a gap in the lattice
   where no path exists.

The guarded rule with unsatisfiable subtype constraints is dead code.

## Example

### Triggering Code

```
language! {
    name: TypedProc;
    primary: Proc;

    types {
        // Lattice: Int and String are incomparable (glb = Bot)
        Top;
        Int  <: Top;
        String <: Top;
        Bot <: Int, String;
    }

    category Proc {
        // Requires x : T where T ≤ Int AND T ≤ String, but glb(Int, String) = Bot
        // and Bot carries no values -- constraint is unsatisfiable for any inhabited type.
        BadCast = "for" "(" "@x" ":" "T ≤ Int AND T ≤ String AND T ≠ Bot" "<-" ch ")" "{" Proc "}";
        Zero = "0";
    }
}
```

The constraint `T ≤ Int AND T ≤ String AND T ≠ Bot` has no solution: the only type
below both `Int` and `String` is `Bot`, which is excluded.

### Diagnostic Output

```
warning[SL01] (TypedProc): subtype constraint set is unsatisfiable (no valid type assignment)
  = note: glb(Int, String) = Bot, excluded by constraint T ≠ Bot
  = hint: check subtype declarations for conflicting edges
```

### Fix

```
// Option 1: Relax the constraint to allow Bot.
FlexCast = "for" "(" "@x" ":" "T ≤ Int AND T ≤ String" "<-" ch ")" "{" Proc "}";

// Option 2: Use a single upper bound instead of incompatible ones.
IntOnly = "for" "(" "@x" ":" "T ≤ Int" "<-" ch ")" "{" Proc "}";

// Option 3: Add a common subtype to the lattice.
// In the types block: Numeric <: Int, String; then constrain T ≤ Numeric.
```

## Rationale

- Subtype constraint satisfaction is decidable for finite lattices: the solver
  computes the glb of all upper bounds and the lub of all lower bounds, then checks
  that glb ≥ lub in the lattice order. For distributive lattices this runs in
  O(|constraints| * |lattice|) time.
- Unsatisfiable subtype constraints indicate a mismatch between the grammar
  author's intended type discipline and the declared type hierarchy. This is a
  common source of silent bugs in polymorphic process calculi.
- The lattice-theoretic check complements the syntactic checks performed by
  unification (UN01) and arithmetic (PB01): subtype constraints operate on the
  type algebra rather than term structure or numeric domains.
