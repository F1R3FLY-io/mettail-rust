# SL02: redundant-subtype-constraint

**Severity:** Note
**Module:** Subtype Lattice
**Feature gate:** `lattice-theory`

## What It Detects

A subtype constraint is already implied by the transitive closure of existing edges
in the declared type hierarchy. The constraint adds no new information to the
solver -- it is derivable from the lattice structure alone.

Detection computes the transitive closure of the subtype relation (via
Warshall-style reachability or topological traversal of the lattice DAG) and checks
whether each explicit constraint is already present as a derived edge.

## Example

### Triggering Code

```
language! {
    name: TypedRouter;
    primary: Proc;

    types {
        Top;
        Number <: Top;
        Int    <: Number;
        Nat    <: Int;
    }

    category Proc {
        // Nat ≤ Number is already implied by Nat ≤ Int ≤ Number
        Redundant = "for" "(" "@x" ":" "Nat ≤ Number" "<-" ch ")" "{" Proc "}";
        Zero = "0";
    }
}
```

The constraint `Nat ≤ Number` is redundant because the lattice already declares
`Nat ≤ Int` and `Int ≤ Number`, and transitivity gives `Nat ≤ Number`.

### Diagnostic Output

```
note[SL02] (TypedRouter): subtype constraint 'Nat ≤ Number' is redundant (implied by transitivity)
  = note: derivation: Nat ≤ Int ≤ Number
  = hint: remove the redundant constraint
```

### Fix

```
// Remove the redundant constraint -- it is already derivable.
Check = "for" "(" "@x" ":" "Nat ≤ Int" "<-" ch ")" "{" Proc "}";

// Or simply omit the subtype guard if the transitive relationship is all you need.
```

## Rationale

- Redundant constraints clutter the grammar specification and may mislead readers
  into thinking the constraint carries independent semantic weight. Removing them
  improves clarity without changing behavior.
- The transitive closure computation runs in O(|V|^3) for the full lattice (via
  Floyd-Warshall) or O(|V| + |E|) per query via BFS/DFS from the source type. For
  typical type hierarchies (tens to hundreds of types) this is negligible.
- In type inference, redundant constraints slow the constraint solver by adding
  unnecessary work to the worklist. While the overhead is small per-constraint, it
  compounds in grammars with many subtype guards.
- This is the lattice-theoretic analogue of PB03 (subsumed arithmetic guard) and
  UN03 (subsumed unification guard): all three detect constraint redundancy in their
  respective domains.
