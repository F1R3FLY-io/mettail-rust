# MT01: multi-channel-overlap

**Severity:** Warning
**Module:** M8 Weighted Multi-Tape Automata
**Feature gate:** `multi-tape`

## What It Detects

- Two tapes in a multi-tape automaton are constrained to identical patterns via `auto_intersect()`, meaning they accept exactly the same label sequences.
- This indicates a redundant channel: the two tapes carry identical information, so one can be eliminated without loss of expressiveness.

## Example

### Triggering Code

```
language! {
    name: DualChannel;
    primary: Proc;

    category Proc {
        // Multi-channel receive where both channels accept the same pattern:
        Sync = "for" "(" "@x" "<-" ch1 "," "@y" "<-" ch2 ")" "{" Body "}";
    }

    // Tape 0 (ch1) and Tape 1 (ch2) are constrained to identical patterns:
    // Both accept: <ident> ("." <ident>)*
    multi_tape_constraint {
        auto_intersect(0, 1);  // forces ch1 and ch2 to share labels
    }
}
```

### Diagnostic Output

```
warning[MT01] (DualChannel): tapes 0 and 1 are constrained to identical patterns (redundant channel)
  = hint: merge the overlapping tapes into a single tape
```

### Fix

```
// Merge the redundant channels into a single channel:
category Proc {
    Sync = "for" "(" "@x" "<-" ch "," "@y" "<-" ch ")" "{" Body "}";
}

// Or differentiate the channels so they carry distinct information:
multi_tape_constraint {
    // No auto_intersect -- let each tape have its own pattern
    tape_0_pattern: <ident>;
    tape_1_pattern: <int>;
}
```

## Rationale

- The lint iterates over `result.overlapping_tapes`, a list of `(tape_idx_1, tape_idx_2)` pairs identified during multi-tape intersection analysis. Two tapes overlap when `auto_intersect(i, j)` constrains them to share the same labels, and the resulting projection of tape `i` equals the projection of tape `j`.
- In the Kempe (2004) multi-tape formulation, `auto_intersect(i, j)` filters transitions to only those where `labels[i] == labels[j]`. If this filter does not reduce the accepted language (i.e., the tapes already produce identical labels on all accepting runs), the constraint is vacuous and the second tape is redundant.
- Redundant tapes increase the multi-tape automaton's state space without adding information. Each tape adds a dimension to the transition space: O(|E_1| x |E_2|) for the `pair()` operation. Eliminating a redundant tape saves quadratic overhead.
- In multi-channel receives `for (@x <- ch1, @y <- ch2)`, overlapping channels mean `x` and `y` always receive the same value. This is likely a specification error -- the programmer probably intended different channels.
