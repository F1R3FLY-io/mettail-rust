# MS02: redundant-feature-check

**Severity:** Note
**Module:** M9 Weighted Automata over Multisets
**Feature gate:** `multiset-automata`

## What It Detects

- All features in the multiset automaton are independent (no feature interactions detected) and all cardinality constraints are trivially satisfied.
- This means the multiset analysis provides no additional information beyond what simpler per-feature analysis would yield. The multi-feature correlation tracking adds overhead without benefit.

## Example

### Triggering Code

```
language! {
    name: Independent;
    primary: Proc;

    category Proc {
        SendA = "a" "!" Msg;
        SendB = "b" "!" Msg;
        RecvA = "a" "?" Msg;
        RecvB = "b" "?" Msg;
    }

    // Features are independent: SendA count doesn't affect SendB count.
    // No cardinality constraints or feature interactions exist.
    multiset_analysis {
        features: [SendA, SendB, RecvA, RecvB];
        // No cardinality constraints specified
    }
}
```

### Diagnostic Output

```
note[MS02] (Independent): all 4 features are independent with trivially satisfied constraints
  = hint: multiset analysis adds no value for independent features
```

### Fix

```
// Option 1: Remove multiset analysis if features are truly independent:
// The multiset-automata feature can be disabled for this grammar.

// Option 2: Add feature interactions or constraints that justify the analysis:
multiset_analysis {
    features: [SendA, SendB, RecvA, RecvB];
    constraint: count(SendA) == count(RecvA);  // send/recv pairing
    constraint: count(SendB) == count(RecvB);
}
```

## Rationale

- The lint fires when `result.num_features > 0 && result.feature_interactions.is_empty() && result.unsatisfiable_constraints.is_empty()`. This triple condition identifies the case where multiset tracking adds no value.
- Feature interactions are detected via `feature_interaction(f1, f2)`: correlated features in accepting runs where one feature's multiplicity depends on another's. Without interactions, each feature can be analyzed independently using simpler counting.
- The multiset semiring operations (pointwise max for `plus`, pointwise add for `times`) become trivially decomposable when features are independent: the joint multiset weight equals the product of per-feature weights.
- In practice, independent features arise when grammar rules produce exactly one feature type per alternative (e.g., each rule contributes to exactly one counter). Correlated features arise in rules that produce multiple feature types simultaneously (e.g., a `Par` rule that contributes to both child feature counts).
- The tropical-over-multisets variant (decidable subclass per Muller et al. 2024) is particularly wasteful for independent features, as it applies tractable approximation to an already-trivial problem.
