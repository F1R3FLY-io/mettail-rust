# MS01: unsatisfiable-cardinality

**Severity:** Warning
**Module:** M9 Weighted Automata over Multisets
**Feature gate:** `multiset-automata`

## What It Detects

- A cardinality constraint on a multiset feature is unsatisfiable: the required multiplicity range `[min, max]` cannot be achieved given the multiset automaton's structure.
- This means the constraint can never be met at runtime, making the guarded branch dead code.

## Example

### Triggering Code

```
language! {
    name: ResourceLang;
    primary: Proc;

    category Proc {
        Par    = Proc "|" Proc;
        Send   = Chan "!" Msg;
        Recv   = Chan "?" Msg;
        Zero   = "0";
    }

    // Cardinality constraint: "at least 5 Send operations in parallel"
    // But the grammar structure limits Par to binary composition,
    // and the maximum reachable Send count is 3 given the input.
    predicate enough_sends: count(Send) >= 5;
    // Multiset analysis determines max multiplicity of Send = 3
    // Constraint [5, *] is unsatisfiable
}
```

### Diagnostic Output

```
warning[MS01] (ResourceLang): cardinality constraint on feature 'Send' ([5, *]) is unsatisfiable
  = hint: relax the cardinality constraint or add more feature-producing rules
```

### Fix

```
// Option 1: Lower the threshold to match achievable multiplicities:
predicate enough_sends: count(Send) >= 2;

// Option 2: Add grammar rules that can produce more Send operations:
category Proc {
    Par    = Proc "|" Proc;
    ParN   = Proc "|" Proc "|" Proc;  // ternary parallel
    Send   = Chan "!" Msg;
    Recv   = Chan "?" Msg;
    Zero   = "0";
}
```

## Rationale

- The lint iterates over `result.unsatisfiable_constraints`, each of which contains a feature name, a minimum bound, and a maximum bound. A constraint is unsatisfiable when the multiset automaton's accepting runs never produce a configuration M where `M(feature) in [min, max]`.
- In the multiset semiring `(N_0^F, max, +, 0, 1)`, the `multiplicity_of(feature, word)` operation counts feature occurrences along accepting paths. The `satisfies_cardinality(constraint)` check determines whether any accepting run meets the bound.
- Unsatisfiable cardinality constraints arise from structural limitations of the grammar: if a feature can only be produced by certain rules, and those rules' reachability limits the maximum multiplicity, then high thresholds become impossible.
- In process algebra, this diagnostic catches resource requirements that the parallel composition structure cannot fulfill -- a common source of deadlocks in Petri net-based analysis (`petri.rs`).
