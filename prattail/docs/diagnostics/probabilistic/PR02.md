# PR02: non-stochastic-state

**Severity:** Warning
**Module:** M7 Probabilistic Automata
**Feature gate:** `probabilistic`

## What It Detects

- The probabilistic automaton has non-normalized transition weights: outgoing transition probabilities from at least one state do not sum to 1.
- A non-stochastic automaton violates the probability axiom sum(P(s -> s')) = 1, which invalidates downstream analyses that assume a proper probability distribution (forward algorithm, Viterbi decoding, entropy computation).

## Example

### Triggering Code

```
language! {
    name: Weighted;
    primary: Stmt;

    category Stmt {
        Assign = <ident> "=" Expr ";"  weight: 0.4;
        If     = "if" Expr "{" Block "}" weight: 0.3;
        While  = "while" Expr "{" Block "}" weight: 0.2;
        // Total: 0.4 + 0.3 + 0.2 = 0.9 != 1.0
    }
}
```

### Diagnostic Output

```
warning[PR02] (Weighted): probabilistic automaton has non-normalized transition weights
  = hint: call normalize() to ensure outgoing probabilities sum to 1
```

### Fix

```
// Option 1: Manually normalize weights to sum to 1.0:
category Stmt {
    Assign = <ident> "=" Expr ";"  weight: 0.45;   // 0.4/0.9 ~ 0.44
    If     = "if" Expr "{" Block "}" weight: 0.33;  // 0.3/0.9 ~ 0.33
    While  = "while" Expr "{" Block "}" weight: 0.22; // 0.2/0.9 ~ 0.22
}

// Option 2: Let the pipeline auto-normalize via normalize().
// The pipeline will call normalize() if is_normalized is false.
```

## Rationale

- The lint fires when `!result.is_normalized`. Normalization status is determined by checking whether per-state log-sum-exp of outgoing transition weights equals 0 (in log-space, log(1) = 0).
- Non-normalized weights cause cascading errors in probabilistic analyses:
  - **Forward algorithm**: P(word) will not sum to 1 over all words, making absolute probabilities meaningless.
  - **Viterbi decoding**: Path comparison is skewed because some states "leak" probability mass.
  - **Entropy computation**: Shannon entropy H = -sum(p * log(p)) requires sum(p) = 1 for correct bits/nats interpretation.
  - **Baum-Welch EM**: Convergence guarantees assume a proper probability distribution.
- The `normalize()` operation applies per-state log-sum-exp normalization in O(|states| x max_out_degree), which is cheap compared to the analysis itself.
- In grammar design, non-stochastic states often indicate an incomplete rule set: the "missing" probability mass corresponds to inputs the grammar does not handle.
