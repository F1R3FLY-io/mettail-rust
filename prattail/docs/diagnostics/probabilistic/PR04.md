# PR04: expected-depth-anomaly

**Severity:** Note
**Module:** M7 Probabilistic Automata
**Feature gate:** `probabilistic`

## What It Detects

- The probabilistic automaton's total selectivity is below 0.01 (1%), meaning the automaton barely accepts any inputs from the expected input distribution.
- This indicates a severe mismatch between the grammar's structure and the input distribution: the rules are collectively very selective, filtering out over 99% of potential inputs.

## Example

### Triggering Code

```
language! {
    name: Specialized;
    primary: Command;

    category Command {
        Deploy  = "deploy" Version Env Config;
        Rollback = "rollback" Version;
    }

    category Version {
        SemVer = <int> "." <int> "." <int>;
    }

    category Env {
        Prod = "production";
        Stage = "staging";
    }

    category Config {
        Json = "{" JsonBody "}";
    }
    // Very narrow grammar: only accepts 2 specific command formats.
    // Total selectivity << 0.01 against a general input distribution.
}
```

### Diagnostic Output

```
note[PR04] (Specialized): total selectivity 0.0042 -- automaton barely accepts any inputs
  = hint: check that grammar rules cover the expected input distribution
```

### Fix

```
// Option 1: Add more rules to broaden coverage:
category Command {
    Deploy   = "deploy" Version Env Config;
    Rollback = "rollback" Version;
    Status   = "status";
    Help     = "help";
    List     = "list" Filter;
}

// Option 2: If narrow coverage is intentional (e.g., a DSL),
// adjust the training corpus to match the expected inputs:
// The selectivity is calculated against the training corpus.
// A corpus of valid commands would yield higher selectivity.
```

## Rationale

- The lint fires when `result.total_selectivity < 0.01 && result.num_states > 1`. Total selectivity is the total language weight computed via `matrix_star()` over the probabilistic automaton, representing the fraction of the input distribution accepted by the grammar.
- The `num_states > 1` guard avoids false positives on trivial single-state automata (which may be intermediate construction artifacts).
- Very low selectivity has practical consequences:
  - **Error recovery**: Most inputs require error recovery, increasing average parse time.
  - **Prediction WFSTs**: The WFST lattice is sparse, reducing the effectiveness of Viterbi decoding.
  - **User experience**: Parsers that reject most inputs feel "broken" to users.
- The term "expected depth anomaly" refers to the interaction between selectivity and recursion: when the probability of entering a recursive rule is very low, the expected recursion depth approaches 0, which may conflict with the grammar's intended recursive structure.
