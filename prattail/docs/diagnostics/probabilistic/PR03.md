# PR03: high-entropy-category

**Severity:** Note
**Module:** M7 Probabilistic Automata
**Feature gate:** `probabilistic`

## What It Detects

- The mean Shannon entropy across grammar categories exceeds 2.0 nats (approximately 2.9 bits), indicating many equally-likely alternatives in the probabilistic model.
- High entropy means the parser faces significant ambiguity: with H > 2.0, the effective number of equally-likely alternatives is e^H > 7.4, so each parsing decision has at least 7+ viable options on average.

## Example

### Triggering Code

```
language! {
    name: Ambiguous;
    primary: Expr;

    category Expr {
        Add    = Expr "+" Expr   weight: 0.1;
        Sub    = Expr "-" Expr   weight: 0.1;
        Mul    = Expr "*" Expr   weight: 0.1;
        Div    = Expr "/" Expr   weight: 0.1;
        Mod    = Expr "%" Expr   weight: 0.1;
        Pow    = Expr "**" Expr  weight: 0.1;
        BitAnd = Expr "&" Expr   weight: 0.1;
        BitOr  = Expr "|" Expr   weight: 0.1;
        Xor    = Expr "^" Expr   weight: 0.1;
        Lit    = <int>           weight: 0.1;
    }
    // H = -10 * 0.1 * ln(0.1) = 10 * 0.1 * 2.302 = 2.302 nats
}
```

### Diagnostic Output

```
note[PR03] (Ambiguous): mean entropy 2.30 nats -- many equally-likely alternatives suggest ambiguity
  = hint: add disambiguation weights or reduce the number of alternatives
```

### Fix

```
// Assign weights reflecting actual usage frequency from corpus data:
category Expr {
    Add    = Expr "+" Expr   weight: 0.25;
    Sub    = Expr "-" Expr   weight: 0.15;
    Mul    = Expr "*" Expr   weight: 0.20;
    Div    = Expr "/" Expr   weight: 0.10;
    Lit    = <int>           weight: 0.30;
    // Remove or gate rarely-used operators behind extensions
}
```

## Rationale

- Shannon entropy H = -sum_i(p_i * ln(p_i)) measures the uncertainty in parsing decisions. The threshold of 2.0 nats corresponds to approximately 7.4 equally-likely alternatives (e^2.0 ~ 7.39).
- The entropy is computed via `EntropyWeight` from the semiring module, which tracks both the probability and the entropy contribution in a single pass using the forward algorithm.
- High entropy grammars cause performance issues in prediction-driven parsers: more alternatives mean more FIRST set entries to check, larger prediction WFSTs, and deeper decision trees in `decision_tree.rs`.
- From an information-theoretic perspective, H > 2.0 means each parsed token conveys less than 2 nats of information about the parse tree structure -- the parser is "guessing" rather than "deciding."
- Corpus-trained weights via `train_from_corpus()` naturally reduce entropy by concentrating probability mass on frequently-used alternatives.
