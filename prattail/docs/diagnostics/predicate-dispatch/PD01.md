# PD01: degenerate-predicate

**Severity:** Warning
**Module:** Predicate Dispatch
**Feature gate:** `predicate-dispatch`
**Source:** `prattail/src/lint.rs` → `lint_pd01_degenerate_predicate()`

## What It Detects

A predicate guard activates no analysis module beyond the base pair (M1 Symbolic
+ M10 Weighted MSO). This indicates the guard is trivially evaluable by the
foundational modules and does not benefit from any specialized analysis.

This is a **structural** warning: it does not mean the predicate is wrong, only
that the dispatch system finds no specialized variety to apply.

## Example

### Triggering Code

```rust
language! {
    pub grammar Trivial {
        category Proc(native_type: "Proc");
        rule PTrue: Proc = "true" {
            // guard: PredicateExpr::True
            // signature: M1 | M10 (base only)
        };
    }
}
```

### Diagnostic Output

```
warning[PD01]: predicate guard activates no specialized module
  signature = 0x0201 (M1_SYMBOLIC | M10_MSO)
  = hint: consider removing the trivially-true guard or adding structural constraints
```

### Fix

Either remove the guard entirely (if it serves no purpose) or add meaningful
structural constraints that would activate specialized modules:
- Add equality tests → activates M6 (Register)
- Add cardinality bounds → activates M9 (Multiset)
- Add cross-channel references → activates M8, M11

## Rationale

Degenerate guards consume parse-time resources for predicate evaluation without
providing useful static analysis feedback. Flagging them helps grammar authors
identify guards that could be strengthened or removed.

## When It Does Not Fire

By Corollary 3.1 of the Dispatch SFA Model, PD01 can only fire for
user-constructed signatures — `extract_features()` always produces signatures
with at least M1 and M10 set. PD01 fires specifically when the signature equals
exactly BASE (0x0201), indicating no specialized module was triggered.
