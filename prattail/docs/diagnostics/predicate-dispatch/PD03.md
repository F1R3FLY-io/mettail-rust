# PD03: dispatch-savings

**Severity:** Info
**Module:** Predicate Dispatch
**Feature gate:** `predicate-dispatch`
**Source:** `prattail/src/lint.rs` → `lint_pd03_dispatch_savings()`

## What It Detects

Reports the number of module invocations skipped by dispatch classification.
This is a positive diagnostic — it confirms the dispatch system is providing
optimization benefit.

## Example

### Diagnostic Output

```
info[PD03]: predicate dispatch skipped 6 of 11 module invocations
  active modules: M1 (Symbolic), M6 (Register), M8 (Multi-Tape),
                  M10 (W. MSO), M11 (Two-Way)
  skipped modules: M2 (Büchi), M3 (AWA), M4 (VPA), M5 (Parity Tree),
                   M7 (Probabilistic), M9 (Multiset)
```

## Interpretation

The savings count reflects the number of `analyze_from_bundle()` calls that
were short-circuited (returned `None`) because the dispatch plan determined
they were irrelevant to the grammar's predicates.

For a grammar with N predicates and K active modules out of 11:
- Modules spawned: K
- Modules skipped: 11 - K
- Savings: (11 - K) × per_module_cost

## When It Does Not Fire

PD03 only fires when at least one module is skipped. If all modules are active
(PD02 condition), PD03 does not fire.

## Related Diagnostics

- **PD01**: Degenerate predicate (max savings: 9 modules skipped)
- **PD02**: All modules activated (0 savings)
