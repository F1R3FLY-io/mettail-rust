# PD02: all-modules-activated

**Severity:** Note
**Module:** Predicate Dispatch
**Feature gate:** `predicate-dispatch`
**Source:** `prattail/src/lint.rs` → `lint_pd02_all_modules_activated()`

## What It Detects

A predicate guard activates all 11 analysis modules (signature = 0x07FF, all
bits set). This means the dispatch provides no optimization benefit — every
module would be spawned anyway.

This is informational: the predicate is complex enough to warrant all analyses.

## Example

### Triggering Code

A predicate combining all morpheme types:

```rholang
letprop balanced x =
  forall_infinite y.
    exists z in {a,b}.
      count(z) >= 1 &&
      related(y) &&        // cross-channel from ch1
      x == last_seen       // equality register
  in
for (@x : balanced <- ch2) { P }
```

### Diagnostic Output

```
note[PD02]: predicate activates all 11 analysis modules (signature = 0x07FF)
  = note: dispatch provides no optimization for this grammar
  = note: all modules will be spawned unconditionally
```

### Fix

This is typically not fixable — the predicate genuinely requires all analyses.
If the grammar is very large, consider splitting it into sub-grammars with
narrower predicate scopes.

## Rationale

PD02 is purely informational. It helps grammar authors understand why dispatch
didn't skip any modules, and may suggest that the predicate could be decomposed.

## Related Diagnostics

- **PD01**: Opposite condition — only base modules activated
- **PD03**: Reports savings (which will be 0 when PD02 fires)
