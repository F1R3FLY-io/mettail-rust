# LT01: logict-search-bound-exceeded

**Severity:** Warning
**Module:** LogicT
**Feature gate:** `logict`

## What It Detects

LogicT's fair interleaving search hit the configured depth limit while solving a
constraint. The theory's `label()` function produced more alternatives than the
`search_bound` parameter allows, causing the search to be truncated before finding
a solution (or exhausting the search space).

This may indicate:

1. **Genuinely deep search space:** The constraint has many valid decompositions
   and the bound is too low for the problem size.
2. **Divergent search:** The constraint generates an infinite stream of
   alternatives (e.g., recursive `CustomMatch` patterns) and the search would not
   terminate without the bound.
3. **Inefficient labeling:** The `label()` function enumerates alternatives in a
   poor order, pushing the solution deep into the search tree.

## Example

### Triggering Code

```
language! {
    name: DeepSearch;
    primary: Proc;

    category Proc {
        // CustomMatch with many alternatives at each choice point
        Match = "for" "(" "@x" ":" custom_match(deep_pattern) "<-" ch ")" "{" Proc "}";
        Zero = "0";
    }
}
```

If `custom_match(deep_pattern)` generates 1,000+ alternatives via `label()` and the
default `search_bound = 1000`, the search is truncated.

### Diagnostic Output

```
warning[LT01] (DeepSearch): LogicT search bound exceeded (limit = 1000, alternatives explored = 1000)
  = note: constraint 'custom_match(deep_pattern)' generated more alternatives than search_bound allows
  = hint: increase the search bound or simplify the constraint
```

### Fix

```
// Option 1: Increase the search bound (if the deep search is expected).
// In the grammar configuration:
config {
    logict_search_bound = 10000;
}

// Option 2: Simplify the constraint to reduce the number of alternatives.
Match = "for" "(" "@x" ":" custom_match(shallow_pattern) "<-" ch ")" "{" Proc "}";

// Option 3: Add constraints to prune the search space early.
Match = "for" "(" "@x" ":" "x > 0 AND custom_match(deep_pattern)" "<-" ch ")" "{" Proc "}";
```

## Rationale

- LogicT's fair interleaving search guarantees that if a solution exists at any
  finite depth, it will eventually be found -- but only if the search bound is
  large enough. The bound exists to prevent compile-time hangs from divergent
  searches.
- The `search_bound` parameter controls the maximum number of `label()`
  alternatives explored across all choice points (not per choice point). This
  provides a global resource limit analogous to a gas meter.
- When the bound is exceeded, the search result is `Unknown` rather than `Unsat`:
  the constraint may still be satisfiable beyond the explored frontier. This is
  why the severity is Warning (possible missed solution) rather than Error
  (definite failure).
- In grammars with nested `CustomMatch` patterns, the search space grows
  exponentially with nesting depth. The bound prevents exponential compile-time
  blowup at the cost of potential incompleteness.

## When It Does Not Fire

- When the search completes within the bound (either finding a solution or
  exhausting the space), no diagnostic is emitted.
- When no LogicT constraints are present in the grammar, the search is never
  invoked.
